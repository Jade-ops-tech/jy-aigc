# journal-editor-ui — 需求规格

## 概述

记录页面：让用户感觉可以立刻、私密地写下一个想法。大面积低干扰的正文输入 + 可选心情标签，支持保存/取消/返回首页、编辑已有记录、确认后删除，以及意外离开时的草稿保留。

## 项目信息

- 项目名: jy-aigc
- 架构类型: pnpm/Turborepo monorepo（React 19 + Vite + TanStack Router + tRPC client）

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |

## 用户故事

- 作为登录用户，我想在几乎无干扰的界面里立刻写下想法并保存。
- 作为登录用户，我想给记录附加一个可选心情，但不被评判。
- 作为登录用户，当我不小心离开页面时，我希望草稿还在。
- 作为登录用户，我想编辑或删除某条旧记录。

## 功能需求

1. [F-001] 记录页为登录后页面，提供大面积、低干扰的正文输入区（Literata），带清晰可访问标签。
2. [F-002] 可选择一个心情标签；固定 6 项（平静/柔软/焦躁/感恩/沉重/期待），视觉柔和、不含评判感、颜色克制且非唯一线索。
3. [F-003] 支持保存、取消并返回首页信息流；系统控件（保存/取消）视觉弱于写作区域。
4. [F-004] 空正文（trim 后）不可保存，提示温和。
5. [F-005] 支持编辑已有记录：从首页卡片进入，预填 body/mood，保存走更新。
6. [F-006] 支持删除记录，需二次确认；确认后返回首页。
7. [F-007] 草稿保留：意外离开（刷新/关闭/路由跳走）后重新进入新记录页可恢复未保存内容。
8. [F-008] 避免厚重边框、密集工具栏与正式格式控制；纯文本，无富文本。

## 非功能需求

- 性能: 输入区无逐字节流请求；草稿写入本地存储做节流。
- 安全: 更新/删除经 tRPC 受保护过程，服务端按 userId 校验所有权。
- 兼容性: 复用 feature 2 的设计系统 token 与 `packages/ui` 控件；mood 枚举复用 `@jy-aigc/api` 的 `MOODS`。

## 验收标准

- [ ] [AC-001] 正文输入区大而低干扰、有可访问标签，聚焦态温暖不刺眼。
- [ ] [AC-002] 可选中/取消一个心情；心情除颜色外有文本标识。
- [ ] [AC-003] 保存空正文被温和阻止；有内容时保存后返回首页并出现在列表。
- [ ] [AC-004] 从卡片进入编辑预填内容，保存走 `journal.update`。
- [ ] [AC-005] 删除有二次确认，确认后经 `journal.delete` 生效并返回首页。
- [ ] [AC-006] 新记录页意外离开后重进可恢复草稿；成功保存后草稿被清除。
- [ ] [AC-007] 所有控件键盘可达、有可访问名称、焦点可见。

## 依赖

- feature 1 `journal-entry-core`：`journal.create` / `journal.update` / `journal.delete` 与 `MOODS`（依赖 `1.T-002`/`1.T-004`/`1.T-005`）。
- feature 2 `journal-feed-ui`：设计系统 token 与卡片入口（依赖 `2.T-001`，编辑入口对接 `2.T-003`）。
- Stitch 页面 `记录碎碎念 (纯净版 Web)`（资源 `projects/16411406785799402988/screens/205f326df16c4d1b808ba5fc4de7d344`）。

## 开放问题

- 草稿保留默认用 localStorage（仅新建场景，按 key 存单条草稿）；如需多草稿或后端草稿请提出。
