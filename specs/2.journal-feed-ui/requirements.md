# journal-feed-ui — 需求规格

## 概述

首页信息流：以固定居中的单列时间线，安静地浏览过往碎碎念，并提供写新记录的主要入口。同时落地整个产品的「Emotional Sanctuary」设计系统（暖色板 + 双字体 + 680px 布局），供本 feature 与记录页共用。

## 项目信息

- 项目名: jy-aigc
- 架构类型: pnpm/Turborepo monorepo（React 19 + Vite + TanStack Router + tRPC client + shadcn/ui）

## 用户故事

- 作为登录用户，我想在首页平静地回看自己的记录，并随手开始写一条新的。
- 作为新用户，当我还没有记录时，我希望看到温和的邀请而不是空洞的空白。

## 功能需求

1. [F-001] 落地设计系统：背景 `#fff8f5`、表面 `#f6ece6/#fcf2eb/#eae1db`、主色 `#7d562d`、主色容器 `#d4a373`、正文 `#1f1b17`、弱化文字 `#50453b`、弱轮廓 `#d4c4b7`；字体 Be Vietnam Pro（UI）+ Literata（日记正文）。
2. [F-002] 首页为登录后页面（`_auth` 保护），展示产品身份与「New Entry」主入口（胶囊按钮，主色 + 暖色阴影）。
3. [F-003] 以日记卡片倒序展示记录：内边距 ≥24px、圆角 16–24px、柔和暖色阴影、正文用 Literata、元数据用紧凑标签样式。
4. [F-004] 卡片展示元数据：日期、时间、心情（心情不可仅靠颜色表达）。
5. [F-005] 记录按天分组；同一天内用一条细微时间线串联；日期标签为紧凑 tag 样式而非大标题。
6. [F-006] 无记录时展示安静、温和的空状态（充足留白、邀请书写的文案）。
7. [F-007] 布局固定居中单列，内容最大宽度约 680px；桌面留宽松留白，移动端左右边距约 20px；日期组间用较大垂直间距。
8. [F-008] 展示 tRPC `journal.list` 的加载与错误状态（克制、温和）。

## 非功能需求

- 性能: 首屏仅拉取列表；字体按需加载，避免 FOUT 影响阅读节奏。
- 安全: 页面在 `_auth` 保护下渲染，未登录重定向登录。
- 兼容性: 沿用 `packages/ui`（shadcn/base-ui）组件与 Tailwind v4 主题。

## 验收标准

- [ ] [AC-001] 设计 token（颜色/字体/680px）在主题中生效，深浅一致于暖色基调。
- [ ] [AC-002] 首页在未登录时重定向到登录，登录后正常渲染。
- [ ] [AC-003] 记录以卡片倒序呈现，按天分组且有紧凑日期标签与细时间线。
- [ ] [AC-004] 无记录时显示温和空状态而非报错/空白。
- [ ] [AC-005] 桌面内容宽度约 680px 居中，移动端左右边距约 20px。
- [ ] [AC-006] 心情信息除颜色外还有文本/图标标识；交互元素可键盘访问、焦点可见。
