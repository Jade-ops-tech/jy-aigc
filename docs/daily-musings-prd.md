# Daily Musings 产品需求文档

## 1. 产品概述

Daily Musings 是一个私密、安静的微型日记产品，用来记录短小的个人想法、情绪和反思。它不应该像一个效率工具，而应该更接近一本亚麻封面的笔记本：安静、有触感、安全，并且刻意降低记录压力。

首个版本应聚焦 Stitch 项目中的两个核心体验：

- 首页信息流：以干净、居中的时间线浏览过往碎碎念。
- 记录页面：用尽量少的阻力写下一条新的想法。

设计来源：Stitch 项目 `Daily Musings`，项目 ID：`16411406785799402988`。

## 2. 产品目标

- 让用户能快速记录转瞬即逝的想法，而不需要管理复杂结构。
- 让回看历史记录的过程保持平静、私人，并带有情绪安全感。
- 通过心情标签和轻量元数据，帮助用户逐渐识别自己的情绪模式。
- 在桌面端和移动端都保持专注的写作与阅读体验。

## 3. 非目标

- 不做公开社交发布、关注、点赞或评论。
- 不做重型任务管理流程。
- 首版不做复杂的数据分析仪表盘。
- 不做协作日记或共享空间。
- 不做复杂富文本编辑，优先保持舒适的纯文本记录体验。

## 4. 目标用户

核心用户是希望拥有一个私密数字空间进行短篇反思的人：学生、创作者、知识工作者，以及任何想收集日常想法但不想承担完整日记系统负担的人。

用户动机：

- 在情绪片段消失前及时捕捉下来。
- 按日期或心情回看过去的想法。
- 保持写作私密且低压力。
- 避免生产力工具带来的忙碌感和压迫感。

## 5. 核心页面

### 首页信息流

Stitch 页面：`首页 - 我的碎碎念 (纯净版 Web)`

资源：`projects/16411406785799402988/screens/816c137aff2d4dfaaf2df7252630c7b5`

首页信息流是用户主要的阅读界面。它应以固定居中的内容列展示记录，并通过舒适的垂直节奏和清晰的日期分组组织内容。

需求：

- 展示产品身份，并提供写下新碎碎念的主要入口。
- 以日记卡片形式展示最近记录。
- 在合适的时候按日期分组。
- 展示记录元数据，例如日期、时间和心情。
- 当没有记录时，提供安静、温和的空状态。
- 桌面端保持舒适的文本行长，内容最大宽度约为 680px。
- 移动端保留充足内边距，页面左右边距约为 20px。

### 记录页面

Stitch 页面：`记录碎碎念 (纯净版 Web)`

资源：`projects/16411406785799402988/screens/205f326df16c4d1b808ba5fc4de7d344`

记录体验应让用户感觉可以立刻、私密地写下一个想法。

需求：

- 提供大面积、低干扰的正文输入区域。
- 允许用户可选地选择一个心情标签。
- 支持保存、取消，并返回首页信息流。
- 系统控件在视觉上应弱于写作区域。
- 避免厚重边框、密集工具栏或让记录变得正式的格式控制。
- 当用户意外离开页面时，支持草稿保留。

## 6. 功能需求

### 记录

- 用户可以创建一条包含正文的新记录。
- 用户可以按倒序时间查看记录。
- 每条记录保存正文、创建时间、可选心情，以及可选标题或首行预览。
- 用户可以编辑已有记录。
- 用户可以在确认后删除记录。
- 空记录不能保存。

### 心情标签

- 用户可以为一条记录附加一个可选心情标签。
- 心情标签应视觉柔和，避免评判感。
- 初始建议心情：平静、柔软、焦躁、感恩、沉重、期待。
- 心情标签不支持用户自定义，首版使用固定选项。
- 心情标签颜色必须保持克制，并在暖色表面色板上具备足够可读性。

### 时间线

- 记录应按天分组。
- 同一天内的记录可以用一条细微的时间线串联。
- 日期标签应采用紧凑的标签样式，而不是很大的章节标题。

### 数据持久化

- 首版必须登录后使用，不提供纯本地匿名记录模式。
- 记录应与已认证用户关联。
- 默认情况下，记录数据只对用户本人私密可见。

### GitHub Token 表单与账户信息

- 用户登录后，可以在表单页面填写个人 GitHub token。
- 系统通过该 token 获取 GitHub 个人账户信息，用于校验 token 有效性并展示基础资料。
- GitHub token 与 GitHub 账户信息应绑定到当前登录用户。
- 一个用户可以绑定多个 GitHub token。
- 用户可以新增、更新或删除自己的 GitHub token 配置。
- 删除 token 配置时，应同步删除或清空与该 token 关联的 GitHub 账户字段。
- 首版只需要读取 GitHub 账户基础信息，不请求仓库、组织、项目或写入权限。
- 推荐用户创建 fine-grained personal access token；如果只调用 `GET /user` 获取基础账户信息，不需要额外权限。
- 如果用户使用 classic personal access token，推荐最小 scope 为 `read:user`；只有当产品需要读取用户邮箱列表时，才额外要求 `user:email`。
- token 属于敏感信息，服务端存储时应加密或使用安全密钥管理方案，前端不得在保存后再次明文展示。
- token 加密采用服务端对称加密：使用环境变量 `GITHUB_TOKEN_ENCRYPTION_KEY` 作为主密钥，不提交到代码仓库。
- 加密密钥长度需满足所选算法要求，首选 AES-256-GCM。
- 数据库存储密文、初始化向量、认证标签和密钥版本，不存明文 token。
- 首版密钥轮换采用版本化方案：新增密钥版本后，新写入 token 使用新版本；旧 token 在用户下次保存、校验或后台迁移时重新加密。
- 拉取 GitHub 信息失败时，应展示明确但克制的错误提示，例如 token 无效、权限不足或网络异常。
- 该功能用于账户集成配置，不应替代产品登录系统。

## 7. 体验原则

- 私密优先：不要暗示分享或公开可见。
- 低压力写作：避免连续打卡、分数或生产力语言。
- 柔和层级：用色调、间距和字体建立层级，而不是强烈边框。
- 舒适阅读：优先保证行长、行高和安静的留白。
- 最少打扰：确认提示和错误提示应温和但清晰。

## 8. 视觉设计要求

设计系统围绕 “Emotional Sanctuary” 展开，即情绪上的安全庇护所。

### 色彩

以 Stitch 设计主题为准：

- 背景色：`#fff8f5`
- 表面容器：`#f6ece6`
- 低层表面容器：`#fcf2eb`
- 最高层表面容器：`#eae1db`
- 主色：`#7d562d`
- 主色容器：`#d4a373`
- 正文文字：`#1f1b17`
- 弱化文字：`#50453b`
- 弱轮廓色：`#d4c4b7`

色彩使用：

- 页面背景使用温暖的奶油色。
- 文字使用炭灰色，避免纯黑。
- 心情和聚焦状态可以少量使用柔和强调色。
- 避免高饱和颜色和强烈渐变。

### 字体

采用双字体策略：

- UI 字体：Be Vietnam Pro，用于导航、按钮、元数据和控件。
- 日记字体：Literata，用于记录正文和更有表达感的写作区域。

Stitch 推荐字体 token：

- 记录展示：Literata，24px 字号，36px 行高。
- 正文内容：Literata，18px 字号，30px 行高。
- UI 正文：Be Vietnam Pro，16px 字号，24px 行高。
- 标签：Be Vietnam Pro，12px 字号，16px 行高，可使用大写/小标题样式。

### 布局

- 使用固定居中的单列布局。
- 内容最大宽度约为 680px。
- 桌面端留出宽松、安静的左右空间。
- 移动端页面边距约为 20px。
- 日期或记录组之间使用较大的垂直间距，暗示时间流逝。

### 组件

日记卡片：

- 内边距不少于 24px。
- 圆角为 16px 到 24px。
- 使用低透明度、暖色调的柔和环境阴影。
- 记录正文使用 Literata。
- 元数据使用紧凑标签样式。

主要操作：

- “New Entry” 按钮应为胶囊形。
- 使用主色，并搭配轻微暖色阴影。
- 在较小屏幕上，必要时可以收起为仅图标的悬浮操作按钮。

输入框：

- 优先使用最小化输入表面。
- 使用底部边框或很浅的填充色。
- 聚焦状态应微妙、温暖，不能刺眼。

空状态：

- 使用充足留白。
- 文案语气应邀请用户书写，而不是让用户觉得自己落后了。

## 9. 可访问性要求

- 所有交互元素都必须可通过键盘访问。
- 按钮和表单字段必须有可访问名称。
- 文本和心情标签颜色需要满足 WCAG 对比度预期。
- 聚焦状态必须可见，并与柔和视觉风格一致。
- 记录页正文输入区域必须有清晰标签。
- 不要只依赖颜色传达心情或状态。

## 11. 成功指标

- 记录创建完成率。
- 创建超过一条记录的用户占比。
- 周回访用户数。
- 平均阅读会话时长。
- 草稿放弃率。
- 保存后删除率，可作为隐私感或记录信心问题的信号。

## 12. 里程碑

### 里程碑 1：原型

- 静态首页信息流，包含示例记录。
- 静态记录页面。
- 本地记录创建流程。
- 响应式桌面端和移动端布局。

### 里程碑 2：私密 MVP

- 已认证用户的数据持久化。
- 记录的创建、读取、更新和删除。
- 心情标签选择。
- 空状态和草稿处理。
- 基础加载和错误状态。
- GitHub token 表单，支持获取账户信息、保存配置和删除配置。

### 里程碑 3：反思增强

- 按文本搜索。
- 按心情筛选。
- 收藏记录。
- 温和的每周反思总结。
- 导出个人档案。

## 13. 待确认问题

当前已确认：

- 首版必须登录后使用。
- 心情标签不允许用户自定义。
- 记录正文保持纯文本，不支持类 Markdown 格式。
- 不提供每日提示。
- 记录页面附近暂不展示额外隐私说明文案。
- GitHub token 首版只读取账户基础信息；fine-grained token 不要求额外权限，classic token 推荐最小 `read:user` scope。
- 允许一个用户绑定多个 GitHub token。
- token 使用服务端环境变量密钥加密存储，首选 AES-256-GCM，并通过密钥版本支持后续轮换。
